# Founder Response

## Project Vision

The goal of this project is to create a simple, trustworthy platform where people can discover and book tools or items available nearby. The focus is on delivering a smooth and reliable core experience rather than building every possible feature in the first version.

---

## Features We Are Building (Keep)

### 1. Home Feed of Nearby Items

This is the core value of the product. Users should immediately see available items near them when they visit the platform.

### 2. Search and Filters

Users can filter items by:

* Distance
* Category
* Free or Paid

This is a minimum requirement and makes the platform practical to use.

### 3. Item Details Page

Each listing will include:

* Photos
* Owner information
* A **Book Now** button

This supports the main discovery-to-booking journey.

### 4. Booking Flow

Users can:

1. Select dates
2. Review details
3. Confirm the booking

This is one of the most important product flows and must be completed.

### 5. Responsive Design and Accessibility

The application will:

* Work well on mobile and desktop.
* Support keyboard navigation.

This ensures a better experience for all users.

### 6. Premium User Experience

"Better than Airbnb" is being interpreted as a design standard, not as a requirement to copy Airbnb's features. The focus is:

* Clean UI
* Strong visual identity
* Simple and enjoyable interactions

---

## Features We Are Reshaping

### Authentication

We will **not** force users to sign up before they can browse listings.

Users can:

* Browse freely
* Search and view items without an account

Users will be required to create an account only when they want to:

* Book an item
* Contact an owner

This approach builds trust and avoids creating unnecessary barriers for new users.

---

## Features We Are Deferring (Future Sprints)

These features are valuable but are not essential for the first release:

* Messaging between owners and borrowers
* Ratings and reviews
* Map view
* Wishlist
* Real-time updates
* Offline support

These features require additional infrastructure and can be implemented once the core experience is stable.

---

## Features We Are Cutting for Version 1

### Fake Urgency Counters

Features like:

> "3 people are viewing this right now!"

will not be implemented unless the data is real.

### Fake Activity or Fake Listings

We will not create artificial activity to make the platform appear busy. Instead, we will use honest empty states such as:

> "Be the first to list a tool nearby."

### Referral Codes

Referral systems are growth features and are unnecessary at this stage.

### Dark Mode

Dark mode is a nice-to-have feature, but development time is better spent improving the core experience.

---

## Product Principles

1. Build trust through honesty.
2. Focus on the core booking journey first.
3. Avoid manipulative design patterns.
4. Deliver a polished and accessible user experience.
5. Add advanced features only after validating the core product.

---

## Final Decision

The first version of the product will prioritise:

* Discover nearby items
* Search and filtering
* Viewing item details
* Booking an item
* Responsive and accessible design
* A premium and trustworthy user experience

Everything else will be considered for future iterations once the core product has been successfully validated.
